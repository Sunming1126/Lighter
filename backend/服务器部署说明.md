# Lighter 服务器部署说明

当前版本只部署 API 基础服务与 PostgreSQL 数据库，真实登录和云同步仍处于关闭状态。

## 第一步：在 Mac 上生成部署包

在 Lighter 项目目录执行：

```bash
cd /Users/zhaoqing/myprojects/Lighter
./backend/scripts/make-deploy-bundle.sh
```

执行后会在项目根目录生成类似文件：

```text
lighter-backend-deploy-20260724T070000Z.tar.gz
```

通过腾讯云控制台的文件上传功能，把该压缩包上传到服务器的
`/home/ubuntu` 目录。

## 第二步：登录服务器并执行部署

```bash
cd /home/ubuntu
tar -xzf lighter-backend-deploy-*.tar.gz
chmod +x backend/scripts/*.sh
./backend/scripts/deploy-server.sh
```

脚本会询问是否继续，输入 `y`。它会自动：

- 检查并安装 Docker 与 Docker Compose。
- 在 `/opt/lighter` 建立按时间编号的发布目录。
- 生成仅 root 可读的数据库随机密码。
- 启动独立 PostgreSQL 和 Lighter API。
- 执行数据库迁移。
- 配置容器异常退出及服务器重启后的自动恢复。
- 等待并验证服务健康状态。

## 查看服务状态

```bash
/opt/lighter/current/scripts/server-status.sh
```

也可以在服务器中直接检查：

```bash
curl http://127.0.0.1:8080/health/live
curl http://127.0.0.1:8080/health/ready
curl http://127.0.0.1:8080/v1
```

## 配置域名与 HTTPS

确认子域名已经解析到服务器后执行：

```bash
chmod +x backend/scripts/configure-domain.sh
backend/scripts/configure-domain.sh
```

脚本默认配置 `lighter.onesay.top`，反向代理到 `127.0.0.1:18080`，并申请
Let's Encrypt HTTPS 证书。脚本会要求输入证书到期提醒邮箱。

首次部署完成后，如果要从公网通过 IP 测试，需要在腾讯云安全组放行
TCP 8080。正式接入 iOS App 前不要直接使用 HTTP IP 地址，还需要配置
域名、HTTPS 和正式的 API 地址。

## 更新后端

重新生成并上传新的压缩包，解压后再次执行：

```bash
./backend/scripts/deploy-server.sh
```

数据库密码和数据卷会被保留，新代码会写入新的发布时间目录。
